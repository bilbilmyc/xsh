# XSH V2 Delivery Plan

> 状态：V2 核心能力已实现；当前进入 macOS 日常使用打磨和 Windows 实机验收阶段。

## V2.0：生产力内核（已完成）

- 命令中心：搜索、分类、标签、收藏、复制、编辑和安全派发。
- 最近连接：仅记录 session ID 与时间，删除会话后自动清理。
- 类 Xshell 底部快捷命令栏：双击空白新增、双击/右键编辑、拖拽排序、分组、搜索、`⌘/Ctrl+1~9` 快捷键。
- 快捷命令支持 `\r`、`\n`、`\t`，发送时自动补回车，不提供密码按钮。
- 高置信度拦截明文密码、Token、私钥、`sshpass -p` 等敏感内容；旧本地存储中的敏感命令会在加载时清理。

## V2.1：终端工作区（已完成）

- 多标签 SSH 连接常驻：切换标签不卸载终端、不主动断开 SSH、不丢失 PTY 程序状态。
- 标签复制、重命名、拖拽排序、锁定/解锁、自定义颜色，以及关闭左侧/右侧/其他/断开/全部标签时保护锁定标签。
- 左右/上下分屏、第二终端选择、命令广播。
- 工作区恢复：安全保存 tab ID、session ID、锁定状态、活动标签和分屏布局；不保存终端输出、命令和凭据。
- 命名工作区：保存、打开、覆盖更新、重命名、删除和密码加密 `.xshworkspace` 导入/导出；打开时为终端生成新 tab ID 并重新建立连接。
- 会话日志、终端搜索、复制全部、清屏、滚到底部、连接状态和手动/自动重连。
- 右键可配置为直接粘贴或操作菜单；选中自动复制与多行粘贴确认均可配置。
- 当前终端支持 `⌘/Ctrl +`、`-`、`0` 临时缩放，并提供工具栏按钮。
- 全局终端字体默认真正覆盖所有会话；可选恢复“会话字体优先”。

## V2.2：认证与网络（已完成核心范围）

- 密码、OpenSSH Key、加密 Key、SSH Agent、keyboard-interactive。
- XSH 自建 SQLite + AES-256-GCM 凭据库，不依赖系统钥匙串。
- TOFU Known Hosts、SHA-256 指纹、Host Key 变化阻断和信任记录管理。
- `~/.ssh/config` 读取与导入、ProxyJump 配置、连接诊断。
- SSH 连接错误按 DNS、超时、拒绝连接、网络不可达、Host Key、密码/私钥认证、私钥读取、协议握手和远端关闭分类，并提供可执行建议。
- 网络诊断支持复制脱敏摘要，不包含主机名、用户名、IP、密码、Credential Ref、私钥路径或 Key Passphrase。
- 本地、远程和动态端口转发及生命周期管理。

## V2.3：SFTP 与会话整理（已完成核心范围）

- 目录浏览、文件与目录递归上传/下载、非空目录递归删除、重命名、新建目录和远程编辑。
- Command/Ctrl 多选、Shift 范围选择、批量下载、批量删除。
- 文件拖拽上传、覆盖/全部覆盖/跳过/全部跳过/重命名冲突策略。
- 上传队列最多两个真实并发任务；worker 等待传输完成后才启动下一项。
- 进度、取消、取消全部、失败/取消重试、单项移除、清理已结束任务。
- 下载使用 `.xsh-part` 和版本化 sidecar 元数据断点续传；仅在远端路径、大小、修改时间一致且断点长度有效时恢复，远端变化后自动重新下载。
- 会话目录树创建、重命名、删除和跨目录拖拽移动。
- 会话支持 Command/Ctrl 多选、Shift 范围选择、Command/Ctrl+A 全选、Esc 清除、批量拖拽/下拉移动、批量收藏/取消收藏、环境/自动重连/标签批量编辑和单次确认批量删除。
- 新建或编辑会话时，对同名会话及相同主机、端口、用户名的连接目标进行保存前提示；不比较或展示任何凭据。

## 安全边界

1. 密码、Key Passphrase 和 Token 只进入 XSH 加密凭据库，不进入系统钥匙串。
2. 会话导出使用一个文件密码加密 `.xshpack`，并携带会话实际引用的登录密码与 Key Passphrase；凭据备份使用独立密码加密的 `.xshbackup`；工作区导出使用密码加密的 `.xshworkspace`。
3. 加密文件内部仍使用 JSON payload，但落盘内容为 AES-256-GCM 密文，不能被记事本直接读取。
4. 导入会先完成格式、密码和完整 payload 校验；`.xshpack` 会恢复其中携带的会话凭据，独立 `.xshbackup` 恢复使用 SQLite 事务写入，失败不会留下半份恢复结果。
5. 工作区、最近连接、命令历史和快捷命令不得保存密码、私钥或终端输出。
6. Host Key 变化默认阻断，不能静默信任。
7. SFTP 文件内容使用流式传输，不把完整文件载入 React 状态；断点文件只保存传输内容和非凭据校验元数据。

## 发布与平台

- macOS：输出免费个人使用的 ad-hoc 签名 `.app` 和 zip，不要求付费开发者账号。
- Windows：构建脚本与 CI 已准备；已完成基础实机验证，默认终端字体使用 Cascadia Mono / Cascadia Code / 系统等宽字体回退栈，不再依赖 Monaco。
